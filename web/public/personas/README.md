# CrossPatch persona portraits

The final five portraits were generated by the product owner with ChatGPT Images and are enabled in `manifest.json`. CrossPatch retains neutral, accessible monograms as the fail-closed fallback when an enabled image cannot load.

Any future replacement must provide both files listed in `manifest.json`:

- `*-source.webp` must be exactly **1024 × 1536** pixels.
- `*.webp` must be an exactly **800 × 1000** central crop of that source.
- Keep the face and defining silhouette inside the central crop. Do not add text, badges, model names, or status colors to the image.
- Keep that asset's `available` field set to `true` in `manifest.json` only after validating both files; the UI reads the manifest and loads the crop automatically. The image renders at exactly **72 × 90** in a seat card and **160 × 200** when expanded.

Required filename pairs are `prosecutor-source.webp` / `prosecutor.webp`, `inspector-source.webp` / `inspector.webp`, `counsel-source.webp` / `counsel.webp`, `magistrate-source.webp` / `magistrate.webp`, and `bailiff-source.webp` / `bailiff.webp`.

Before enabling an asset, call `validateReplacement` from `components/PersonaPortrait.tsx` and leave `available` set to `false` when it returns `false`. The helper reports source or crop dimension mismatches; toggling the manifest does not perform an automatic runtime rejection.
